## 대표 페르소나

### P-01 첫 실전 면접을 준비하는 사용자

- 상황: 면접이 일주일 남았고 예상 질문 답안을 문서로 작성했다.
- 행동: 답안을 외운 뒤 혼자 말해 보지만 시선과 말버릇을 확인하지 않는다.
- 목표: 답변이 막혀도 자신의 경험으로 다시 구성하고 싶다.
- 핵심 시나리오: 이력서 업로드 → 기술 면접관 선택 → 장비 확인 → 5문항 면접 → 근거 확인 → 취약 질문 재연습.

### P-02 경험을 직무 언어로 바꾸려는 사용자

- 상황: 비전공 경험을 개발·데이터 직무 역량으로 연결해야 한다.
- 행동: 채용 공고 키워드와 자신의 프로젝트를 수동으로 대조한다.
- 목표: 면접관이 납득할 수 있도록 역할·행동·결과를 구조화하고 싶다.
- 핵심 시나리오: 지원 기업·직무 입력 → HR/기술 면접 비교 → 답변 내용 근거 확인 → 개선 답변으로 재연습.

### P-03 반복 연습의 효과를 확인하는 사용자

- 상황: 두 번 이상 모의면접을 완료했지만 개선 방향이 분산됐다.
- 행동: 점수만 비교하고 실제 행동 변화는 기록하지 않는다.
- 목표: 이번 주에 고칠 한 가지를 선택하고 다음 결과에서 확인하고 싶다.
- 핵심 시나리오: 대시보드 성장 추이 → 취약 축 선택 → 같은 조건 재연습 → 이전·현재 근거 비교.

## 핵심 JTBD

1. 면접이 가까워졌을 때, 실제와 비슷한 순서로 말해 보며 당황을 줄이고 싶다.
2. 답변이 약하다고 느낄 때, 어느 문장과 행동이 문제인지 확인해 다음 답변을 바꾸고 싶다.
3. 반복 연습할 때, 동일한 기준으로 변화를 비교해 개선이 실제로 일어났는지 알고 싶다.
4. 새로운 직무에 지원할 때, 내 경험을 해당 직무가 이해하는 언어로 설명하고 싶다.
