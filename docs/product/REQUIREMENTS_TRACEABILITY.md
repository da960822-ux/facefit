## 3. 화면설계서와 PRD 차이

| 화면설계·소개 자료의 표현 | PRD 반영 | 이유 |
| --- | --- | --- |
| React + Vite | V1 프론트엔드 스택으로 전환 완료 | 기존 화면·행동·URL을 보존한 React Router SPA |
| Claude 특정 모델명 | OpenAI-compatible LLM 어댑터 | 공급자·버전 종속을 줄이고 검증 후 선택 |
| 실시간 피드백 표시 | 면접 중 코치 정보는 최소화, 정밀 평가는 사후 제공 | 면접 몰입과 지연·인지부하를 보호 |
| 감정 분석 | 종합 점수에서 제외 | 관측 가능성과 공정성 우선 |
| 내 목소리 AI 클론 | P2 선택형 베타 | 동의·삭제·오용·품질 검증 필요 |
| 연속 실시간 면접 | 반이중 P0, 수동/전이중 인터럽트 P1 | 안정적인 A/V 동기와 턴 완료를 먼저 검증 |
| CV 백그라운드 병렬 처리 | 사후 비동기 분석으로 분리 | GPU·CPU 부하가 실시간 대화를 방해하지 않게 함 |

## 5. 업데이트 규칙

- 기능 상태를 `Done`으로 바꾸려면 PRD 수용 기준과 QA 시나리오를 통과해야 한다.
- UI가 존재한다는 이유만으로 실제 기능 완료로 표기하지 않는다.
- 요구사항 ID가 변경되면 이 문서, PRD, 로드맵을 같은 변경에서 함께 갱신한다.
