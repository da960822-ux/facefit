## 공통 원칙

- 오류 문구는 발생한 일, 사용자 영향, 가능한 행동 순서로 작성한다.
- 재시도해도 입력 데이터가 사라지지 않아야 한다. 보장 불가 시 미리 알린다.
- 실제 상태가 없으면 성공처럼 보이는 목업 상태를 운영에 사용하지 않는다.
- 기술 오류 코드와 사용자 문구를 분리하고 원본 음성·답변을 기술 로그에 남기지 않는다.

## 복구 우선순위

1. 같은 화면에서 안전하게 재시도한다.
2. 입력·답변을 유지한 채 대체 장치·텍스트·음성·정적 화면으로 전환한다.
3. 세션을 안전 종료하고 분석 또는 대시보드에서 재조회한다.
4. 복구 불가 시 보존된 범위와 문의에 필요한 식별자만 안내한다.

## 미결정 사항

- 분석 지연 판정 시간, 재시도 횟수, 세션 복구 가능 시간
- 카메라 없이 음성 전용 면접을 허용할지
- 업로드 취소와 부분 업로드 정리 방식
