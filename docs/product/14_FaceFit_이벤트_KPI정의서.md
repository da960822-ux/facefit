| 항목 | 내용 |

## 서비스 목표와 지표 후보

| 목표 ID | 서비스 목표 | 북극성 지표 후보 | 성공 지표 | 가드레일 |
| --- | --- | --- | --- | --- |
| KPI-OBJ-01 | 실제 면접 연습 완료 | 주간 완료 면접 세션 수 | 시작→완료율, 5문항 완료율 | 중도 종료율, 세션 오류율 |
| KPI-OBJ-02 | 근거를 이해한 개선 | 근거가 있는 리포트 조회 세션 수 | 근거 열람률, 개선안 재생/확인률 | 일부 축 실패율, 결과 이의 피드백 |
| KPI-OBJ-03 | 반복 개선 | 7일 내 재연습 사용자 수 | 재연습률, 2회 이상 완료율 | 과도한 알림·강제 CTA 없음 |
| KPI-OBJ-04 | 신뢰 가능한 운영 | 오류 없이 완료된 세션 비율 | 권한 복구율, 분석 완료율 | 민감정보 로그 0건, 무동의 음성 복제 0건 |

목표값은 운영 기준선과 표본 수가 없어 `[미정]`이다. 첫 측정 주기에는 분포와 이탈 구간을 확인하고 이후 목표를 승인한다.

| 6 | 면접 시작 | `interview_started` | 첫 질문 턴 진입 |

## 행동·품질 이벤트

| 이벤트 ID | 이벤트명 | 발생 조건 | 주요 속성 | 측정 목적·활용 |
| --- | --- | --- | --- | --- |
| EV-001 | `landing_viewed` | SC-001 렌더 | 유입 경로 범주, 익명 세션 | 진입 기준선 |
| EV-002 | `auth_started` | 로그인·가입 제출 시작 | 방식 [미정], 화면 ID | 인증 이탈 분석 |
| EV-003 | `auth_completed` | 인증 성공 | 신규/기존, 목적지 | 가입 전환 |
| EV-004 | `resume_upload_completed` | 업로드 성공 | 형식, 크기 구간, 처리 시간 | 업로드 사용성 |
| EV-005 | `resume_upload_failed` | 업로드 실패 | 오류 범주, 재시도 여부 | 정책·파서 개선 |
| EV-006 | `interview_setup_completed` | 설정 유효·다음 이동 | 면접관, 강도, 이력서 사용 여부 | 설정 선호·완료율 |
| EV-007 | `permission_decided` | 장치 권한 결과 | 카메라/마이크, 허용/거부 | 권한 이탈·복구 |
| EV-008 | `equipment_check_completed` | 장비 확인 완료 | 장치 가용 범주, 소요 시간 | 준비 퍼널 |
| EV-009 | `voice_profile_decided` | 완료·건너뛰기 | 선택 결과, 실패 경험 여부 | 선택 기능 가치 |
| EV-010 | `interview_started` | 첫 질문 진입 | 면접관, 강도, 폴백 단계 | 시작 기준선 |
| EV-011 | `answer_completed` | 답변 저장 완료 | 질문 순서, 완료 방식 버튼/Space/VAD, 처리 시간 | 턴 UX·VAD 검증 |
| EV-012 | `avatar_fallback_activated` | 폴백 전환 | 원인 범주, 이전/이후 단계 | 실시간 품질 |
| EV-013 | `interview_ended` | 세션 종료 | 정상/중도/오류, 완료 문항 수 | 중도 종료 분석 |
| EV-014 | `analysis_status_changed` | 분석 단계 변경 | 이전/현재 단계, 경과 구간 | 병목·지연 |
| EV-015 | `analysis_failed` | 분석 실패 | 단계, 오류 범주, 재시도 | 품질·복구 |
| EV-016 | `report_viewed` | 리포트 첫 조회 | 세션, 부분 결과 여부 | 결과 도달 |
| EV-017 | `evidence_opened` | 축·질문 근거 열람 | 축, 근거 유형 | 설명 가능성 사용 |
| EV-018 | `retry_started` | 재연습 새 세션 | 진입 화면, 이전 설정 사용 [협의] | 반복 개선 |
| EV-019 | `dashboard_viewed` | 대시보드 렌더 | 기록 수 구간, 분석 중 존재 | 성장 기능 사용 |
| EV-020 | `recoverable_error_shown` | 사용자 복구 가능 오류 | 화면·상태·오류 범주·복구 행동 | 오류 UX 개선 |

## 이벤트 원칙

- 사용자 식별자는 분석용 가명 키를 사용하고 이력서·질문·답변 원문, 파일명, 음성·영상은 속성에 넣지 않는다.
- 이벤트는 시도·성공·실패를 구분하고 화면 ID·요구사항 ID와 연결한다.
- 클라이언트 시간만 신뢰하지 않고 서버·실시간 서비스 시간과 비교할 수 있어야 한다.
- 중복 전송과 재시도 정책은 데이터 담당과 협의한다.

## 미결정 사항

- 북극성 지표 최종 선택과 측정 주기
- 인증 사용자·익명 세션 연결 방식과 데이터 보존
- 재연습의 동일 조건 정의와 성장 지표 산식

- 기준선 수집 후 KPI 목표값과 가드레일 임계를 결정한다.
