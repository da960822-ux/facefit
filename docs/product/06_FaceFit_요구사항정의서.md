## 표기 규칙

- 우선순위: P0는 V1 출시 차단, P1은 V1 안정화 또는 직후, P2는 검증 후 고도화다.
- 상태: `확정`은 현재 UI·자료와 범위가 합의된 항목, `가정`은 검증 전 설계, `협의 필요`는 담당자 결정이 필요한 항목이다.
- 데이터·API 스키마는 확정하지 않는다. 관련 문서는 [11 데이터 요구사항](11_FaceFit_데이터요구사항.md), [12 연동 요구사항](12_FaceFit_연동요구사항_초안.md)이다.

| FR-012 | 녹음 중이고 입력 요소·모달에 포커스가 없을 때 | 버튼이나 Space를 연속 실행하면 | 완료 동작은 한 번만 처리된다. |
| FR-013 | VAD가 침묵 후보를 보낼 때 | 자동 완료 정책을 적용하면 | 수동 완료가 계속 가능하고 오탐 복구 규칙을 따른다. |
| FR-014~016 | 세션 종료 후 | 분석·조회·재연습을 선택하면 | 상태 손실 없이 리포트 또는 새 설정으로 이동한다. |
| NFR-001~004 | 지원 환경에서 | 핵심 플로우와 장애 복구를 시험하면 | 13의 확정 기준을 충족하고 측정 로그가 남는다. |
| UIR-001~003 | 비동기 상태가 변할 때 | 화면을 관찰하면 | 실제 상태와 가능한 행동만 표시되고 허위 수치는 없다. |
| A11Y-001~002 | 키보드·스크린리더·모션 감소 환경에서 | 핵심 플로우를 수행하면 | 조작·상태·포커스를 인식할 수 있다. |
| SEC-001~003 | 민감 데이터 처리 전후 | 수집·조회·삭제를 수행하면 | 동의·권한·보존 정책을 적용하고 무단 노출을 차단한다. |
| LOG-001~002 | 퍼널·오류 이벤트가 발생할 때 | 로그가 전송되면 | 목적에 필요한 속성만 기록하고 원본 답변·음성은 제외한다. |
