## 1. 감사 범위와 근거

확인 우선순위는 실제 실행 코드, 저장소 문서, 동일 프로젝트의 PDF·HTML, 이전 요청 기록 순이다. 요청에 명시된 `05_약식_기획서(이원준).hwp`, `InterviewAI_화면흐름도_PDR.md`, `Face Fit UI - Standalone4.html`은 현재 저장소와 첨부 경로에서 찾지 못했다. 따라서 이름이 유사한 자료를 원본으로 간주하지 않고 아래 대체 근거만 사용했다.

- 실제 구현: `src/pages`, `src/components/facefit`, `src/lib/facefit-content.ts`
- 제품 문서: `PRD.md`, `OKR.md`, `MVP_ROADMAP.md`, `REALTIME_AVATAR_ARCHITECTURE.md`, `REQUIREMENTS_TRACEABILITY.md`
- 화면 근거: `FaceFit_UIUX_멘토링_화면설계서.pdf`, `uiux멘토링화면설계서.html`
- 서비스 근거: `FACE-FIT_ 설명 가능한 AI 면접 코칭 서비스.pdf`
- 참고 HWP: `프롬프트용 기획서.hwp`는 AirNote 문서여서 FaceFit 사실 근거로 사용하지 않고 문서 구성 방식만 참고했다.

## 3. 중복, 충돌, 오래된 내용

| 유형 | 발견 내용 | 영향 | 처리 결정 |
| --- | --- | --- | --- |
| 중복 | PRD, 로드맵, 추적표가 동일 P0 기능을 반복한다. | 변경 시 문서 불일치 | 요구사항은 06, 일정 우선순위는 17, 연결은 16을 단일 기준으로 한다. |
| 해소 | 프론트는 React+Vite로 전환됐고 기존 화면·행동·URL을 React Router SPA로 보존했다. | 프레임워크 전환에 따른 직접 URL 접근·정적 배포 회귀가 발생할 수 있다. | Vercel rewrite와 Nginx SPA fallback을 적용하고 전환 검증을 지속한다. |
| 충돌 | 결제·이용권 제외 원칙과 `/pricing` 발표용 화면이 공존한다. | V1 범위 오해 | `/pricing`은 SC-012, 구현됨·V1 제외·결제 없음으로 표기한다. |
| 충돌 | 초기 11개 화면과 현재 `/signup`, `/pricing`이 추가로 존재한다. | 화면 수 불일치 | 가입은 핵심 흐름에 포함하고, 요금제는 비핵심 콘셉트로 격리한다. |
| 오래됨 | PDF의 PostgreSQL 제안은 확정 백엔드와 일치한다고 확인되지 않았다. | 기술 확정 오해 | 데이터베이스 제안으로만 보존하고 데이터·API는 미정 처리한다. |
| 오래됨 | 정적 면접관 이미지는 실시간 아바타 목표와 다르다. | 완성도 오해 | 현재는 목업, 목표는 OpenAvatarChat+MuseTalk 연동으로 구분한다. |
| 충돌 | 5초 침묵 자동 완료 문구가 있으나 실제 VAD 연동은 없다. | 사용자가 자동 동작을 기대 | 수동 `답변 완료`를 주 방식, 침묵 감지는 보조·연동 전으로 정의한다. |

## 4. 개발 착수 전 누락

- 화면·기능·데이터·연동·QA의 공통 ID 체계
- 권한 거부, 장치 없음, 업로드 실패, 분석 지연, 세션 만료의 복구 규칙
- 파일 형식·용량, 보존·삭제, 음성 프로필 동의 정책의 확정 주체
- 프론트엔드와 AI·백엔드 사이의 상태 계약과 타임아웃 협의 항목
- 접근성, 브라우저, 로깅, 개인정보의 검증 가능한 기준
- 포트폴리오에서 문제·대안·결정·결과를 연결하는 서술

## 6. 제작 우선순위

1. 06 요구사항, 07 기능, 08 화면, 09 상태를 확정해 개발 공통어를 만든다.
2. 12 연동 경계와 13 비기능 기준으로 프론트·AI·백엔드 협의를 연다.
3. 15 QA와 16 추적표로 누락을 차단한다.
4. 17 백로그와 18 의사결정 기록으로 실행 순서를 합의한다.
5. 19 케이스 스터디는 검증된 사실과 추후 입력을 분리한다.
