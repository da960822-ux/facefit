# FaceFit 연동 요구사항 초안

| 항목 | 내용 |
| --- | --- |
| 목적 | 실제 API 명세 이전에 프론트엔드와 외부·AI·백엔드 사이의 책임과 상태 경계를 정의한다. |
| 대상 독자 | 프론트엔드, AI·백엔드, PM, QA |
| 버전 | 1.0 |
| 작성일 | 2026-07-20 |
| 상태 | 초안·엔드포인트와 JSON 미정 |

이 문서는 API 명세서가 아니다. URL, 메서드, 응답 JSON, 데이터베이스 필드, 제공자별 SDK는 기술 협의 후 확정한다.

## 용어 경계

- VAD: 음성 활동과 침묵 후보를 감지한다.
- ASR/STT: 음성을 인식해 텍스트와 필요한 경우 시간 정보를 만든다.
- TTS: 질문·개선 답변 텍스트를 음성으로 합성한다.
- LLM: 질문·꼬리 질문·피드백 서술을 생성한다.
- CV: 시선·자세 등 영상 관찰 특징을 분석한다.
- OpenAvatarChat: 실시간 대화 모듈을 오케스트레이션한다.
- MuseTalk: 면접관 음성과 영상의 립싱크 아바타 출력을 담당한다.

## 연동 등록부

| 연동 ID | 기능 목적 | 요청 주체→제공 주체 | 필요한 입력 | 필요한 출력 | 트리거 | 프론트 표시 상태 |
| --- | --- | --- | --- | --- | --- | --- |
| INT-001 | 인증·권한 | 웹→인증/앱 서버 | 계정 입력·목적 경로 | 인증 상태·사용자 참조·오류 범주 | 로그인·가입·재진입 | 로딩·성공·실패·세션 만료 |
| INT-002 | 파일 업로드·문서 처리 | 웹→파일/문서 서비스 | 파일·목적·사용자 동의 | 업로드·검증·처리 상태와 문서 참조 | 파일 선택 | 업로드·성공·형식/용량/통신 실패 |
| INT-003 | VAD | 실시간 서비스→VAD | 사용자 오디오 스트림 | 발화 시작·종료·침묵 후보 | 사용자 턴 | 대기·녹음·침묵 후보·오류 |
| INT-004 | ASR/STT | 실시간/분석→ASR | 오디오·언어·세션 참조 | 전사·상태·품질 메타 [협의] | 실시간 또는 사후 분석 | 인식 중·완료·실패 |
| INT-005 | LLM | 앱/실시간/분석→LLM | 이력서·직무·질문·답변 맥락 최소값 | 질문·피드백 서술·안전 상태 | 질문 생성·분석 | 생성 중·완료·안전 차단·실패 |
| INT-006 | TTS·선택 음성 | 실시간/리포트→TTS | 텍스트·음성 옵션·동의 참조 | 오디오·상태 | 질문·개선 답변 재생 | 합성·재생·오류·기본 음성 폴백 |
| INT-007 | 실시간 아바타 | 웹↔OpenAvatarChat/MuseTalk | 세션·TTS 오디오·면접관 자산 | RTC 영상·음성·연결 상태 | 면접 시작·각 질문 | 연결·발화·정적/음성 폴백·오류 |
| INT-008 | CV 분석 | 분석 서비스→CV | 사용자 영상 또는 특징 스트림 [협의] | 시선·자세 관찰 구간과 상태 | 면접 중/종료 후 [협의] | 사용자 화면에는 분석 후 결과만 표시 |
| INT-009 | 분석 오케스트레이션 | 웹/앱→분석 시스템 | 세션 참조·분석 대상 | 5단계 상태·완료·지연·실패 | 세션 마감 | 분석 중·지연·실패·완료 |
| INT-010 | 리포트·기록 조회 | 웹→앱/리포트 서비스 | 사용자·세션·조회 조건 | 리포트·기록·성장 상태 | 결과·대시보드 진입 | 로딩·성공·부분 결과·빈 상태·실패 |

## 처리 요구사항

| 연동 ID | 로딩·성공 처리 | 실패·타임아웃 처리 | 재시도 | 협의 필요 항목 |
| --- | --- | --- | --- | --- |
| INT-001 | 버튼 잠금, 성공 후 목적지 이동 | 계정 정보 노출 없는 오류, 만료 시 재인증 | 사용자 재시도 | 인증 방식·토큰·게스트 |
| INT-002 | 파일명·처리 단계 유지 | 원인 범주와 대체 입력 제공 | 네트워크 오류는 가능 | 형식·용량·취소·파서 결과 |
| INT-003 | 녹음 UI와 활동 상태 동기화 | 수동 완료 유지, VAD 오류로 세션 중단 금지 | 세션 내 재초기화 [협의] | 침묵 시간·자동 확정·오탐 처리 |
| INT-004 | 전사는 사용자 답변과 같은 턴에 연결 | 사후 재처리 가능 여부 표시 | 분석용은 정책에 따라 | 실시간/사후 모델, 타임코드 |
| INT-005 | 생성 중 중복 요청 차단 | 고정 안전 질문 또는 재시도 [협의] | 제한적 | 프롬프트·금칙어·컨텍스트 길이 |
| INT-006 | 재생 상태와 중지 제공 | 기본 TTS 또는 텍스트 폴백 | 가능 [협의] | 음성 제공자·동의·캐시 |
| INT-007 | 연결 상태를 명시하고 미디어 재생 | 영상→음성+정적→텍스트 순 폴백 | 자동 재연결 횟수 협의 | RTC 방식·동시성·세션 TTL·GPU 용량 |
| INT-008 | 처리 단계만 표시 | 해당 축 부분 실패로 표시, 다른 축 유지 | 사후 재처리 [협의] | 실시간/사후, 원본 저장, 공정성 |
| INT-009 | 실제 단계만 표시, 이탈 허용 | 지연·실패와 재조회 경로 | 작업 멱등성 필요 | 상태 목록·지연 임계·재처리 |
| INT-010 | 리포트 버전·부분 상태 유지 | 빈 상태와 조회 실패 구분 | 조회 재시도 | 페이지네이션·버전·삭제 반영 |

## 책임 경계

| 영역 | 프론트엔드 | AI·백엔드 |
| --- | --- | --- |
| 입력·표시 | 입력 검증, 권한 UI, 상태·오류·폴백 표시 | 서버 검증, 상태·오류 범주 제공 |
| 면접 턴 | 완료 입력 잠금, 현재 턴 UI, 미디어 재생 | 질문·답변 상태, VAD/ASR/TTS/아바타 오케스트레이션 |
| 분석 | 5단계와 재조회 표시 | CV·ASR·LLM 처리, 상태·근거·버전 관리 |
| 보안 | 토큰·민감값 비노출, 브라우저 권한 안내 | 인증·인가, 암호화, 보존·삭제·감사 로그 |

## 미결정 사항

- 모든 연동의 실제 인터페이스, 오류 코드, 타임아웃, 멱등 키
- OpenAvatarChat 배포 경계와 MuseTalk 동시 처리 용량
- 분석 작업의 부분 성공·재처리·리포트 버전 계약

## 다음 협의 항목

- 프론트·AI·백엔드가 INT-003~010의 상태 전이 워크숍을 진행한다.
- 확정된 계약은 별도 API 명세에서 관리하고 본 문서에는 링크만 추가한다.
