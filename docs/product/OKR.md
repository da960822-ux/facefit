# Face Fit MVP OKR

> 기간: 2026-07-20 ~ 2026-09-30  
> 상태: 목표안  
> 원칙: 현재 수치는 실측 기준선이 아닌 MVP 목표값이며, 첫 2주 측정 후 조정한다.

## Objective 1. 실제 면접처럼 이어지는 실시간 아바타 면접을 완성한다

| Key Result | 목표 | 측정 방법 |
| --- | --- | --- |
| KR 1.1 | OpenAvatarChat + MuseTalk로 5문항·15분 면접 E2E 성공 | 자동 시나리오 및 내부 세션 로그 |
| KR 1.2 | 치명적 중단 없는 세션 비율 95% 이상 | 20회 이상 내부 세션 |
| KR 1.3 | 발화 종료→면접관 음성 시작 P50 2.5초 이하, P95 4초 이하 | 턴별 분산 트레이싱 |
| KR 1.4 | 오디오·영상 동기 오차 절대값 150ms 이하 | 녹화 샘플 프레임·파형 비교 |
| KR 1.5 | 답변 종료 오검출률 5% 이하 | 자동 종료와 사용자 수정 이벤트 비교 |

### 선행 지표

- RTC 연결 성공률
- Avatar 준비 시간과 캐시 적중률
- ASR 첫 부분 전사 지연
- 수동 답변 완료 비율
- 정적 이미지 + 음성 폴백 발생률

## Objective 2. 사용자가 점수의 이유와 다음 행동을 이해하는 리포트를 제공한다

| Key Result | 목표 | 측정 방법 |
| --- | --- | --- |
| KR 2.1 | 완료 리포트 100%에 4축 근거·신뢰도·개선 행동 포함 | 리포트 스키마 검증 |
| KR 2.2 | 분석 완료 시간 P95 3분 이하 | 분석 작업 큐 메트릭 |
| KR 2.3 | 사용자 테스트 참여자의 80% 이상이 다음 연습 행동을 1개 이상 정확히 설명 | 10명 이상 과업 테스트 |
| KR 2.4 | 전문가 샘플 평가와 4축 등급 일치율 75% 이상 | 블라인드 이중 평가 |
| KR 2.5 | 신뢰도 부족 구간을 점수로 오표시한 비율 0% | 저조도·가림·무음 테스트셋 |

### 가드레일

- 감정 추정, 외모, 성별, 연령, 장애 추론을 점수에 사용하지 않는다.
- 루브릭 버전이 다른 결과를 단순 증감으로 비교하지 않는다.
- 근거가 없는 LLM 서술을 리포트에 노출하지 않는다.

## Objective 3. 사용자가 한 번의 체험에서 반복 연습으로 이어지는 가치를 확인한다

| Key Result | 목표 | 측정 방법 |
| --- | --- | --- |
| KR 3.1 | 장비 점검 시작 대비 면접 완료율 70% 이상 | 퍼널 이벤트 |
| KR 3.2 | 완료 사용자 중 7일 내 재연습률 30% 이상 | 계정·세션 코호트 |
| KR 3.3 | 리포트 완료 후 24시간 내 열람률 85% 이상 | `report_viewed` 이벤트 |
| KR 3.4 | 리포트 열람자의 40% 이상이 추천 연습 또는 같은 조건 재면접을 시작 | CTA 전환 이벤트 |
| KR 3.5 | 사용자 만족도 5점 만점 평균 4.0 이상 | 세션 후 3문항 설문 |

## Objective 4. 민감한 영상·음성 데이터를 신뢰할 수 있게 운영한다

| Key Result | 목표 | 측정 방법 |
| --- | --- | --- |
| KR 4.1 | 동의 없는 영상·음성·음성 프로필 저장 0건 | 저장 이벤트와 동의 로그 대조 |
| KR 4.2 | 사용자 삭제 요청 테스트 100% 통과 | 원본·파생 데이터 삭제 점검 |
| KR 4.3 | 운영 로그의 원문 답변·원본 생체정보 노출 0건 | 로그 정적 검사·샘플 감사 |
| KR 4.4 | 연결 끊김·분석 실패·GPU 포화의 폴백 시나리오 100% 통과 | 장애 주입 테스트 |
| KR 4.5 | 사용 OSS·모델·부속 모델 라이선스 검토 완료 | 의존성 및 모델 카드 목록 |

## 운영 리듬

- 매주 월요일: KR 추이, 주요 장애, 다음 실험 검토
- 매주 수요일: 실시간 파이프라인 지연·GPU 사용량 리뷰
- 매주 금요일: 사용자 세션 2회 이상 관찰, 리포트 품질 샘플링
- 2주마다: 목표값과 범위 재검토. 목표 변경 시 이유와 전후 값을 문서에 남긴다.

## 완료 판정

MVP는 Objective 1과 2의 모든 KR, Objective 4의 KR 4.1~4.4를 충족해야 외부 사용자 테스트로 전환한다. 성장·재방문 KR이 미달이면 기능 추가보다 온보딩, 지연, 리포트 이해도를 먼저 개선한다.
