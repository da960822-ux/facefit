# FaceFit 정책 및 비즈니스 룰

| 항목 | 내용 |
| --- | --- |
| 목적 | 기능 전반에 반복 적용되는 운영·표시·데이터 규칙을 구분해 관리한다. |
| 대상 독자 | PM, UX, 개발, QA, 보안·개인정보 담당 |
| 버전 | 1.0 |
| 작성일 | 2026-07-20 |
| 상태 | 확정·협의 필요 혼합 |

## 정책 등록부

| 정책 ID | 주제 | 규칙 | 상태 | 근거·영향 |
| --- | --- | --- | --- | --- |
| POL-001 | 서비스 범위 | 데스크톱 웹 모의면접 연습이 V1이며 결제·이용권·관리자·모바일 앱은 제외한다. | 확정 | 프로젝트 고정 범위 |
| POL-002 | 지원 파일 형식 | 현행 UI는 PDF·DOCX를 표시한다. 실제 파서 지원 여부 확인 후 확정한다. | 협의 필요 | FN-002, SC-004 |
| POL-003 | 파일 용량 | 현행 UI는 최대 10MB를 표시한다. 인프라·보안 검토 전 정책값으로 확정하지 않는다. | 협의 필요 | FN-002, ST-014 |
| POL-004 | 업로드 실패 | 입력을 유지하고 실패 이유, 재시도, 다른 파일, 텍스트 입력 경로를 제공한다. | 확정 | UIR-001 |
| POL-005 | 면접 문항 | V1 기본은 5문항, 예상 15~20분이다. 질문 구성 규칙은 AI 담당과 협의한다. | 가정 | 현재 UI·기획서 |
| POL-006 | 답변 종료 | `답변 완료` 버튼과 Space가 주 방식이며 하나의 완료 처리로 중복을 막는다. | 확정 | FR-012 |
| POL-007 | 침묵 감지 | VAD는 보조 기능이다. 오탐 위험 때문에 수동 완료를 항상 제공한다. 자동 확정 조건은 협의한다. | 부분 확정 | FR-013 |
| POL-008 | 면접 중 종료 | 확인 모달에서 영향·저장 상태를 알리고 계속 또는 종료 후 분석을 선택한다. | 확정 | ST-023 |
| POL-009 | 결과 공개 | 분석 완료 전 결과를 성공처럼 표시하지 않는다. 부분 결과 공개 여부는 협의한다. | 협의 필요 | FN-015~016 |
| POL-010 | 분석 실패 | 답변 보존 여부와 재처리 가능 여부를 표시하고 리포트를 임의 생성하지 않는다. | 확정 | ST-019 |
| POL-011 | 리포트 재조회 | 본인 소유의 완료 리포트는 대시보드에서 재조회한다. 보존 기간은 협의한다. | 부분 확정 | SEC-002 |
| POL-012 | 점수 표기 | 4축 점수에는 근거·관찰 범위·개선 행동을 연결한다. 감정·성격·합격 가능성을 평가하지 않는다. | 확정 | BR-002, BR-004 |
| POL-013 | 성장 비교 | 동일 정의의 축만 비교하고 2회 미만에는 데이터 부족을 표시한다. 비교 집단은 임의 생성하지 않는다. | 확정 | BR-003 |
| POL-014 | 재연습 | 새 세션을 만들며 이전 기록을 덮어쓰지 않는다. 설정 복제 범위는 협의한다. | 부분 확정 | FN-018 |
| POL-015 | 개인정보 안내 | 수집 전 목적·필수/선택·보존·삭제를 알린다. 이력서·음성·영상은 민감 데이터로 취급한다. | 확정 | SEC-001 |
| POL-016 | 영상·음성 처리 | 실시간 처리와 저장을 구분한다. 저장 필요성과 보존 기간은 최소화 원칙으로 협의한다. | 협의 필요 | DT-007~010 |
| POL-017 | 음성 프로필 | 선택 기능이며 별도 동의 없이 음성 복제하지 않는다. 건너뛰어도 핵심 면접·리포트를 이용한다. | 확정 | SEC-003 |
| POL-018 | 삭제 | 사용자가 삭제를 요청할 수 있어야 한다. 삭제 범위·완료 기한·백업 반영은 협의한다. | 협의 필요 | 개인정보 담당 |
| POL-019 | 아바타 폴백 | 영상 실패 시 음성+정적 면접관, 음성 실패 시 질문 텍스트 순으로 핵심 흐름을 유지한다. | 가정 | OpenAvatarChat 설계 |
| POL-020 | 안내 문구 | 진단·합격 단정, 근거 없는 정확도·남은 시간·퍼센트 표현을 금지한다. | 확정 | BR-004, UIR-003 |
| POL-021 | 요금제 | `/pricing`은 발표용 콘셉트이며 실제 결제·구독·이용권 상태를 만들지 않는다. V1 핵심 IA에서 제외한다. | 확정 | 현행 코드와 범위 충돌 정리 |
| POL-022 | 프론트 연동 | API 명세 전 엔드포인트·응답 JSON·DB 필드를 프론트 문서에서 확정하지 않는다. | 확정 | 역할 분리 원칙 |
